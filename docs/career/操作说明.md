## 里面有什么

| 文件名 | 用途 | 谁看 |
|---|---|---|
| `CLAUDE.md` | 给 Claude 读取的目录上下文，包含你的背景、求职方向、项目说明 | Claude |
| `操作说明.md` | 本文件，说明这个文件夹怎么用 | 你 |
| `简历改造方案.md` | Markdown 格式的简历改造方案 | Claude / 你 |
| `简历改造方案.docx` | Word 格式的简历改造方案，方便阅读整理 | 你 |

## 你应该怎么使用这个文件夹

### 1. 阅读核心文档

先看 `简历改造方案.docx`，了解当前简历问题和改造计划。

### 2. 按行动计划执行

方案里有三个月行动计划，建议你按周执行：

- **6 月**：确定简历主线，整理项目到 GitHub
- **7 月**：给项目加测试/日志/文档，准备面试
- **8 月**：投递简历，面试复盘

### 3. 每次找 Claude 时指定这个文件夹

在 Claude Code 里，你可以说：

> "读取我 ai-demos 项目 docs/career 文件夹里的内容，帮我继续改造简历。"

这样 Claude 会先读 `CLAUDE.md`，快速了解你的背景和进度，不用每次重新解释。

### 4. 更新进度

完成一个任务后，可以在这个文件夹里新建文档记录进度，例如：

- `项目进度_饥荒MOD_Agent.md`
- `面试复盘_2026-06-XX.md`
- `简历_v1.md`

保持更新，方便 Claude 掌握最新状态。

### 5. 生成新文档时的规范

- 长文档优先让 Claude 生成 Word（.docx）格式
- 临时草稿可以用 Markdown（.md）格式
- 不要直接往桌面乱放文件，先放到 `ai-demos/docs/` 下的对应子目录
- 最终定稿的简历、项目 README 等材料，建议再分类到各自项目目录

## 联系 Claude 的常用说法

- "帮我更新简历改造方案"
- "根据我的求职资料，帮我准备 XX 公司的面试"
- "帮我看看 GitHub 项目 README 怎么写"
- "把这次面试复盘记录到 ai-demos/docs/career 文件夹"

## 注意事项

- `CLAUDE.md` 是给 Claude 看的，你可以不读它，但也不要删除。
- 如果你换了求职方向，记得让 Claude 更新 `CLAUDE.md`。
- 这个文件夹里的内容会持续发展，建议你每周整理一次。
